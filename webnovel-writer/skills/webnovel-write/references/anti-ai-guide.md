# Anti-AI 写作指南

> 核心理念：写的时候就写对，比写完再改好。
> 加载时机：Step 2 起草正文前。

## 你最容易犯的错

作为 LLM，你有以下根深蒂固的倾向。写作时必须刻意对抗：

1. **每段写完整闭环**——你倾向于“起因→经过→结果→感悟”四段走完。刻意删掉感悟，留未解决的余味。
2. **用副词修饰一切**——你倾向于写“缓缓开口”“淡淡说道”“微微点头”。删掉副词，用具体动作替代。
3. **所有角色同一套反应**——你倾向于让每个人都“瞳孔微缩”“心中一凛”。给每个角色设计专属微动作。
4. **对话像辩论赛**——你倾向于让角色完整、理性地表达观点。让角色说话带潜台词，言行不一，有省略和打断。
5. **情绪贴标签**——你倾向于写“他感到愤怒”。改为生理反应+微动作，让读者自己判断。
6. **信息均匀分布**——你倾向于每段放差不多的信息量。刻意制造疏密对比，有的段落只有一句话。
7. **安全着陆**——你倾向于在章末把所有冲突解决。刻意留下至少一个未解决的问题。
8. **展示后解释**——你倾向于用动作展示后，再补一句解释。删掉解释句，信任读者的理解力。

## 写作时的 5 个即时检查

每写完一个段落，快速过一遍：

1. **闭环检查**：这段是否走完了“起因→经过→结果→感悟”？→ 删掉最后的感悟/总结句
2. **副词检查**：最近 3 个动作描写是否用了“缓缓/淡淡/微微/轻轻”？→ 删掉副词，换成不同感官通道的细节
3. **情绪检查**：角色的情绪是“告诉读者”还是“展示给读者”？→ 把“他感到X”改为生理反应+微动作
4. **对话检查**：这句对话是在推进冲突还是在解释背景？→ 删掉背景解释，加入角色的真实意图
5. **节奏检查**：最近 500 字的节奏是否均匀？→ 插入一个短句爆发点、一段沉默、或一个突然的动作

## 替代方案速查表

| AI 癖好 | 替代方向 | 示例 |
|---------|---------|------|
| “他感到愤怒” | 生理+微动作+决策 | “指节捏得发白，舌尖尝到铁锈味。再等一秒他就动手。” |
| “缓缓说道” | 删副词，加前置动作 | “他把杯子搁下——'你确定？'” |
| “眸中闪过一丝X” | 个性化行为 | “她咬了下嘴唇内侧”（紧张） / “他开始拧笔帽”（焦虑） |
| “心中暗道” | 直接写内心句 | 删掉“心中暗道”四个字，后面的内容直接保留 |
| 四段闭环 | 截断在高点 | 删掉最后的感悟句，让读者自己体会 |
| 全员书面语 | 口语化+个人口癖 | 加口头禅（“得了吧”）、省略（“走？”“嗯。”）、打断 |
| 段末总结 | 删总结句 | 用动作或对话结尾，不写“他终于明白了” |
| 展示后解释 | 删解释句 | 写完“她把门摔上”后不要再写“她显然很生气” |
| “更重要的是……” | 直接说事 | 删掉递进模板词，直接给信息 |
| 情绪即时切换 | 加过渡 | “他压下怒意，但声音还是哑了半拍” |

## 对话专项

好的对话特征：
- 有人抢话、有人沉默、有人答非所问
- 角色不会把真实想法全盘托出
- 不同角色有不同的说话节奏（有人啰嗦、有人惜字如金）
- said tag 不超过 30%，优先用前置动作（“他把烟掐灭——'行。'”）

坏的对话特征（你最容易写的）：
- A说→B说→A说→B说，严格交替
- 每句都完整回应上一句
- “XX说道”“XX淡淡道”贯穿全篇
- 角色说话像在做报告

## 节奏专项

好的节奏特征：
- 有的段落只有一句话，有的段落有十几句
- 紧张时句子变短、变碎
- 平静时可以有长段描写
- 信息密集的段落和留白的段落交替出现

坏的节奏特征（你最容易写的）：
- 每段 3-5 句，长度均匀
- 信息密度恒定，无峰谷
- 转场全靠“时间飞逝”“不知不觉”

## Round 19 Phase X1 · 前 5 章 reader-critic 写前自检清单

> **触发**：当 chapter ∈ (1, 2, 3, 4, 5)，writer 在 Step 2A 起草前必须按下表逐项自检并写入 `tmp/pre_draft_self_check_ch{NNNN}.json`。这是与全卷 reader-critic <75 P0 硬阻止配套的“写前预防”，目的是把 reader-critic 历史 Ch3=62 / Ch4=58 的谷底类问题前置堵掉。

### 5 类前置自检项（每项 PASS / WARN / FAIL）

#### 1. 金手指首披露时序

- 本章是否符合 state.json 的 golden_finger 披露梯度？
- 不超前披露下一卷剧情；不延后兑现已承诺爽点
- FAIL 触发：超前 1+ 卷或延后 ≥ 5 章兑现

#### 2. 突兀编号 / 系统术语铺垫

- 章内出现的所有数字编号（#4732 / D-25 / 23:42 / 0% 8% 等）是否本章前文已铺垫？
- 命名实体（陈默 / 林晚秋 / 苏蕊 / 老李 / 秦岳等）首次出现是否有 1 句以上的身份暗示？
- FAIL 触发：≥ 2 个数字编号或命名实体无铺垫直接抛出

#### 3. 大纲爽点兑现密度

- 本章 outline 列出的 N 条爽点 / 钩子，正文兑现率 ≥ 70%？
- 关键爽点（reader-pull-checker 标“strong”）必须 100% 兑现
- FAIL 触发：兑现率 < 50% 或关键 strong 钩子缺兑现

#### 4. 伏笔铺设节奏

- 本章新铺设的钩子线条数 ≤ 2（避免 Ch3 一次铺 5 条新期待线的雷）
- 上章未闭合钩子若 ≥ 5 条，本章必须先闭合 ≥ 1 条再开新钩
- FAIL 触发：本章新开 ≥ 3 条钩子且上章未闭 ≥ 5 条

#### 5. 读者卡点检查

- 章内是否有 ≥ 1 处“读者读到会停下来 google / 翻前章 / 出戏”的点？
- 候选风险：突兀专有名词、复杂金手指机制单段说明、跨语境隐喻
- FAIL 触发：发现 ≥ 2 处读者卡点

### 与 reader-critic-checker 的耦合

- 自检 `verdict=REWRITE_RECOMMENDED`（≥2 FAIL）→ writer **必须**回 Step 1 重做大纲再起草
- 自检 `verdict=NEEDS_ADJUST`（1 FAIL 或 ≥3 WARN）→ writer 起草时刻意规避，并在 writing_guidance.constraints 追加 “X1 自检命中：{item_id}” 提醒
- 自检 `verdict=PASS` → 正常起草

### 与 polish_cycle 的耦合

polish_cycle 完成后 reader-critic-checker 复测：

- 若 reader-critic 仍 < 75 → polish 必须再来一轮（不能 commit）
- 若 reader-critic ≥ 75 但 < 80 且本章是前 5 章 → polish_log notes 追加“Ch{N} reader-critic={score}（前 5 章警告区）”，下章自检时强提醒
