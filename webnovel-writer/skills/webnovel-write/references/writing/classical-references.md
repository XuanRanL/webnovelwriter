<context>
网文引经据典的核心矛盾：用典能显著提升文化厚度和世界真实感，但用不好就是"炫学"、"百度百科贴"。
本指南解决三个问题：(1) 引用什么 (2) 怎么融入不生硬 (3) 多少是"恰到好处"。
</context>

<instructions>

## 一、核心原则

1. **典故不是装饰品，是世界的骨头**。好的引用让读者觉得"这个世界有根"，而不是"作者在秀学识"。
2. **化用 > 引用 > 注释**。把典故融进描写和对话里，读者自己发现出处时的惊喜感远强于直接标注。
3. **宁缺毋滥**。一卷50章只用10处精准引用，效果远超每章硬塞3处。
4. **大纲阶段规划，章节阶段执行**。引用在大纲/详细大纲中以"引用锚点"标注，写章时由 Context Agent 推荐，Step 2A 按需融入。临时硬写容易生硬。
5. **典故即伏笔**。最高级的引用是读者第一遍觉得"嗯有文化"，回头看才发现"这句话早就在暗示结局"。

## 二、引用类型与密度等级

| 类型 | 密度等级 | 融入难度 | 说明 |
|------|---------|---------|------|
| 题材核心典籍 | A（世界骨架） | 低 | 在世界观里就是"教科书/物理定律"，自然出现在教材/课堂/推演中。如：修仙文的道经、命理文的《三命通会》、法律文的判例法条 |
| 哲学/思想 | B（角色内化） | 中 | 通过角色领悟呈现，不直接引用原文。读者自己发现出处时有惊喜感 |
| 诗词意象 | C（氛围点缀） | 高 | 化用半句融入环境描写或内心独白。全卷最多3-4次。一句点睛 > 整首引用 |
| 历史典故/人物 | C（世界厚度） | 中 | 不直接引用历史人物，让世界内角色/事件隐约对应。读者自己联想到时觉得"有根" |
| 原创诗词/口诀 | 特殊（最高优先） | 低 | 小说自己的名句比借用任何经典都有力。应提前设计，在设定集中管理 |
| 互联网梗/现代文化 | D（社交载体） | 中 | 只通过第三方载体（微博/弹幕/论坛/同学闲聊）呈现，主角不说梗 |

## 三、密度控制（硬规则）

| 维度 | 推荐值 | 上限 |
|------|--------|------|
| 全卷（50章）总引用 | 10-15 处 | 20 处 |
| 单章引用 | 0-1 处 | 2 处 |
| 章间间隔 | 3-5 章 | 最密不低于 2 章 |
| 互联网梗/卷 | 3-5 处 | 8 处 |
| 原创口诀首次出场间隔 | 每条间隔 10+ 章 | — |

**允许不用**：Context Agent 推荐后，Step 2A 判断"本章不适合引用"可以跳过。40 章无引用是正常的。

## 四、融入技法（按类型）

### 4.1 题材核心典籍（A级）

直接作为世界规则呈现，不需要标注出处——在这个世界里，经典就是"教科书"。

**正确示范**：
```
教官在黑板上写下八个字："金空则鸣，火空则明。"
```

**错误示范**：
```
正如古人在《三命通会》中所言："金空则鸣，火空则明。"
（×：没人在物理课上说"正如牛顿在《自然哲学的数学原理》中所言"）
```

### 4.2 哲学/思想（B级）

让角色"悟到"而非"背诵"。

**正确示范**：
```
季空看着空白命盘，脑海中闪过一个念头——空的东西，反而能装下所有。
（读者自己联想到《道德经》"有生于无"）
```

**错误示范**：
```
季空想起老子说过"天下万物生于有，有生于无"，觉得和自己的处境很像。
（×：说明书，把读者的思考空间堵死了）
```

### 4.3 诗词意象（C级）

化用意境，不引原文。让叙述替角色说话。

**正确示范**：
```
他推开宿舍门。走廊空到只剩值班室那一格灯光。
（读者自然联想"千山鸟飞绝"的孤独感）
```

**错误示范**：
```
走廊空无一人，让他想起柳宗元那句"千山鸟飞绝，万径人踪灭"。
（×：网文不是高考阅读理解）
```

### 4.4 互联网梗（D级）

**硬规则**：
1. 只选存活 > 1 年的经典梗，拒绝月抛型流量梗
2. 只通过第三方载体呈现——微博评论、弹幕、论坛帖、同学闲聊
3. 永远不从主角嘴里说出——会毁人设
4. 不解释梗——懂的人会心一笑，不懂的人不影响阅读

### 4.5 原创口诀/名言

**设计原则**：
- 提前在设定集中设计完整的口诀/歌谣/铭文
- 每条有明确的"世界内来源"（民谣/碑刻/教材/角色台词），不凭空出现
- 分批释放：首次只给 1-2 句，后续章节逐步补全。**不完整 > 完整**——读者的好奇心是最好的钩子
- 口诀可以承载伏笔：表面是世界设定，实际暗示关键剧情

## 五、"典故即伏笔"技法

最高级的引用不是装饰，而是伏笔载体。操作方法：

1. **选一句多义的经典**：表面意思适合当前场景，深层意思指向未来真相
2. **第一次出现时不解释**：让读者觉得"嗯，有道理"然后翻页
3. **关键章节时让角色/读者重新理解**：同一句话在不同语境下含义翻转
4. **在伏笔追踪中双重登记**：既在典故引用库标注，也在伏笔追踪的"典故伏笔"分类登记

**示例**：
> "有支而无干，曰亡"
> - 首次出现（Ch5 课堂）：教科书定义，读者记住"空亡=没有天干"
> - 二次理解（Ch15 首次填实）：读者恍然——"有支无干"意味着"有容器没限制"，这就是主角能力的理论根基
> - 终极理解（Ch950）：原来"无干"不是缺陷，是自由

## 六、项目设定集模板

每个小说项目建议创建以下文件（非强制，按需）：

### `设定集/典故引用库.md`（推荐）

```markdown
# 典故引用库

## 使用规则
1. 全卷总量上限：10-15 处。单章上限 2 处。
2. 允许不用——Context Agent 推荐后可跳过。
3. 原创 > 借用——同等效果优先用原创口诀。

## 一、{题材核心典籍}（密度A）
| # | 原文 | 出处 | 融入载体 | 适用场景 | 伏笔功能 |
|---|------|------|---------|---------|---------|
| M01 | ... | ... | 教材/课堂 | ... | 无/伏笔→Ch? |

## 二、{哲学/思想}（密度B）
...

## 三、{诗词意象}（密度C）
...

## 四、{原创口诀}（→ 详见 原创诗词口诀.md）
...

## 五、{互联网梗}（密度D）
...

## 第N卷引用规划总表
| 章 | 引用内容 | 类型 | 载体 | 伏笔 |
```

### `设定集/原创诗词口诀.md`（推荐）

世界内的原创歌谣、铭文、口诀、角色标志性台词。每条含：
- 世界内来源（谁写的/刻在哪/从哪传来）
- 全文 + 逐句伏笔解析（仅作者可见）
- 分批释放规划（首次只给几句，后续补全）
- 使用约束（总次数/间隔/专属角色）

### 大纲中的"引用锚点"字段

在详细大纲的章节条目中添加：
```
- 引用锚点: M01"原文"（载体，伏笔说明）/ 无
```

## 七、工作流集成点

| 步骤 | 动作 | 条件 |
|------|------|------|
| Step 0 | 检查 `设定集/典故引用库.md` 是否存在 | 不存在时提示用户"建议创建典故引用库以提升文化质感" |
| Step 1（Context Agent） | 读取引用库 + 原创口诀，检查本章大纲是否有引用锚点，输出 0-2 条引用建议（含载体+融入方式） | 引用库存在时必读；不存在时跳过 |
| Step 2A（起草） | 按 Context Agent 建议融入正文。可判断不适合而跳过 | 加载本指南（审查命中引用相关问题时） |
| Step 3（审查） | 引用自然度作为文笔质感/设定一致性的检查维度 | 自动 |
| Step 4（润色） | 修复"引用生硬/炫学感/出处错误"类问题 | 审查报告命中时加载本指南 |

## 八、常见错误与修复

| 错误 | 问题 | 修复方向 |
|------|------|---------|
| "正如XX所言" | 说教感，打断叙事节奏 | 删掉引导语，让角色/环境自然呈现 |
| 整段引用原文 | 掉出小说变百科 | 化用 1-2 句，融入描写 |
| 主角说梗 | 毁人设 | 改为配角/社交媒体载体 |
| 每章都引用 | 炫学感 | 删到全卷 10-15 处 |
| 引用与剧情无关 | 装饰品感 | 删掉，或改为承载伏笔 |
| 注释出处 | 论文感 | 删掉注释，题材核心典籍不需要标出处 |
| **AI 记忆的诗词字词错误** | **幻觉风险** | **Search tool 实时验证（见第九节）** |
| **AI 记忆的作者张冠李戴** | **幻觉风险** | **Search tool 交叉验证** |
| **使用过期的互联网热梗** | **时效性失败** | **Search tool 查询当前年份时效性** |

## 九、Search tool 强制使用规范（防 AI 幻觉 + 时效性）

> **为什么必须**：AI 记忆的典故、诗词、民俗常有字词错误、作者错误、时代错位等幻觉。互联网热梗时效性强，2 年前的梗现在可能过时。本项目追求"最高质量"的典故系统**不能依赖 AI 记忆**——必须通过 search tool 实时验证。

### 9.1 必须调用 search 的场景

| 场景 | Skill 位置 | 查询模板 | 失败时的降级 |
|---|---|---|---|
| **init 创作原创诗词前** | init Step 5.6 | `"{题材} {关键意象} 诗词 意境"` 避免撞车 | 跳过该原创条目 |
| **init 建立诗词典故池时** | init Step 5.6 | `"{诗词首句} 出自 {作者}"` 验证字词 | 不登记该条目 |
| **init 建立民俗典故库时** | init Step 5.6 | `"{地域} {民俗元素} 典故 出处"` 验证真实性 | 标记"待核实" |
| **init 建立互联网梗白名单时** | init Step 5.6 | `"{当前年份} 互联网热梗 网络用语"` 时效性 | 只用经典梗 |
| **write 融入冷门引用前** | write Step 2A | `"{诗词首句} {作者} 原文"` 字词校对 | 跳过该次引用 |
| **data-agent 抽到 unknown 时** | data-agent Step B.5 | `"{snippet} 出处 诗词"` 补全元数据 | 标记 `needs_manual_review` |
| **review 发现可疑引用时** | review / audit | `"{原文} 真实 出处"` 交叉验证 | 标记为疑似 AI 幻觉 |

### 9.2 无需调用 search 的场景（信任 AI 记忆）

1. **顶级知名诗词**：苏轼《江城子》"十年生死两茫茫"、白居易《梦微之》、李白/杜甫顶流作品 —— AI 记忆足够准确
2. **原创资产**：本项目 `设定集/原创诗词口诀.md` 里的内容 —— 信任本项目数据
3. **主角标志性台词**：作者决定，不需要外部验证
4. **已多次验证过的条目**：若典故引用库的条目有 `verified_at` 时间戳 < 30 天，跳过重复验证

### 9.3 工具优先级（按用户 memory 偏好）

1. **Tavily Search MCP**（首选）—— 用户明确偏好，见 memory `feedback_use_tavily.md`
2. **Tavily Research MCP**（复杂验证，需要多源交叉对比时）
3. **Tavily Extract MCP**（验证具体 URL 的内容准确性时）
4. **WebSearch**（Tavily 失败时的降级）

### 9.4 搜索查询模板库

```
# 模板 1：诗词字词验证
query = "{诗词首句} 作者 {claimed_author} 原文"
max_results = 3

# 模板 2：民俗典故真实性
query = "{region} {custom_name} 民俗 传说 出处"
max_results = 5

# 模板 3：互联网热梗时效性
query = "2026 {meme_name} 网络用语 最新"
time_range = "month"  # 近一个月内的引用

# 模板 4：角色文学图腾匹配
query = "{character_archetype} 适配 诗词 {emotion_theme}"
max_results = 5
include_domains = ["gushiwen.cn", "gushiwen.org", "sou-yun.cn"]

# 模板 5：原创诗词撞车检查
query = '"{original_draft_first_line}"'  # 完整引号强制精确匹配
max_results = 3
```

### 9.5 中文搜索规则（用户 memory 强制）

> 见 user memory `feedback_search_in_chinese.md`：中文小说项目中所有搜索必须用中文，禁止用英文。

- ✅ 正确："苏轼 江城子 十年生死两茫茫 原文"
- ❌ 错误："Su Shi Jiang Cheng Zi original text"

### 9.6 每章强制搜索（user memory）

> 见 user memory `feedback_force_tavily_search.md`：每章至少搜索 1 次，不得跳过，普通章也要搜场景类型写作技巧。

- write Step 0.5 或 Step 2A 必须包含至少 1 次 Tavily 调用
- 即使本章不引用任何典故，也要搜索场景/情绪/写作技巧相关内容
- 搜索结果记录到 `调研笔记/章节参考/ch{NNNN}.md`

### 9.7 搜索结果的登记

验证通过的引用必须在典故引用库中登记 `verified_at`、`verification_source`、`verification_snippet`：

```markdown
| # | 原文 | 出处 | ... | verified_at | verification_source |
|---|---|---|---|---|---|
| S01 | 蓼蓼者莪 | 诗经·蓼莪 | ... | 2026-04-10 | gushiwen.cn/... |
```

这样下次写作时可以跳过重复验证，节约 search 调用次数。
