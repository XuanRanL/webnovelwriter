# dialogue-checker (对话质量检查器)

> **职责**: 对话质量审查专家，检测对话辨识度、意图层次、信息倾倒与独白失控问题。

> **输出格式**: 遵循 `${CLAUDE_PLUGIN_ROOT}/references/checker-output-schema.md` 统一 JSON Schema

## 检查范围

**输入**: 单章或章节区间（如 `45` / `"45-46"`）

**输出**: 对话质量分析、辨识度评估、意图层次分布、问题定位与修复建议。

## 执行流程

### 第一步: 加载上下文

**输入参数**:
```json
{
  "project_root": "{PROJECT_ROOT}",
  "storage_path": ".webnovel/",
  "state_file": ".webnovel/state.json",
  "chapter_file": "正文/第{NNNN}章-{title_safe}.md"
}
```

`chapter_file` 应传实际章节文件路径；若当前项目仍使用旧格式 `正文/第{NNNN}章.md`，同样允许。

**并行读取**:
1. `正文/` 下的目标章节
2. `{project_root}/.webnovel/state.json`（角色语音规则）
3. 相关角色卡（获取角色说话风格设定）
4. `设定集/` 下角色相关文件（语言习惯、身份背景）

### 第二步: 对话提取与分段

**提取所有对话段落**:
1. 识别直接引语（引号内内容）
2. 识别间接引语（"他说"/"她道"等引导的内容）
3. 标注每段对话的说话者
4. 计算对话占全文比例（`dialogue_ratio`）

**健康范围**: `dialogue_ratio` 建议 0.25–0.55
- < 0.20：对话过少，叙述偏重，角色互动不足
- > 0.60：对话过多，可能缺少动作/环境描写

### 第三步: 辨识度检查（核心）

**遮住人名测试**: 对每段对话，遮住说话者名字后判断：
- 能否通过句式、用词、语气分辨说话者？
- 是否有角色独特的语言标记（口头禅、句式偏好、专业术语）？

**检查维度**:

| 维度 | 标准 | 问题信号 |
|------|------|----------|
| 句式长度 | 不同角色应有不同的平均句长 | 所有角色都用相似长度的句子 |
| 语气词 | 角色应有独特语气词/口头禅 | 全员标准书面语 |
| 专业度 | 不同背景角色用词专业度不同 | 底层角色说文绉绉的话 |
| 信息密度 | 不同角色单句信息量不同 | 所有角色都完整表达，无省略 |
| 情绪表达 | 不同性格角色表达情绪方式不同 | 全员直抒胸臆 |

**输出**:
- `distinguishable_voices`: 可辨识的独立声音数
- `indistinguishable_pairs`: 遮住人名后无法区分的角色对数（>0 即需改写）

### 第四步: 意图层次检查

**每段对话必须有意图**（试探/施压/回避/诱导/防御/安抚/威胁/请求）：
- 纯信息传递（"说明书式对话"）→ 标记为 `DIALOGUE_INFODUMP`
- 对话只是为了让读者知道某个设定 → 标记为 `DIALOGUE_INFODUMP`
- 对话完整解释背景且逻辑过满 → 标记为 `DIALOGUE_INFODUMP`

**意图类型统计**: `intent_types` 列表
- 健康对话应有 ≥ 2 种意图类型
- 全员同一种意图（如全是"解释"）= 单调

### 第五步: 独白控制检查

**单人连续独白字数** (`longest_monologue_chars`):
- ≤ 150 字：正常
- 150-200 字：注意
- > 200 字：标记为 `DIALOGUE_MONOLOGUE`，建议打断

**内心独白泛滥检查**:
- 连续 3+ 段内心独白且无外部动作/对话打断 → 标记 `PADDING` 问题

> **权责划分**: 对话检查器关注**局部结构**（连续堆积），密度检查器关注**全局比例**（`inner_monologue_ratio > 0.25`）。两者互补，详见 `checker-output-schema.md` "内心独白检查权责划分"。

### 第五步半: 潜台词检测

**`subtext_instances`**: 识别表面说A实际意图B的对话

**检测规则**:
1. 扫描每轮对话，判断说话者的**字面含义**与**真实意图**是否存在差异
2. 识别信号：
   - 话语与上下文语境不匹配（如紧张局面中说轻松的话）
   - 角色已知有隐藏信息/目的
   - 对话后跟随暗示真实意图的动作描写（如"说着，手不自觉握紧了剑柄"）
   - 对方的反应暗示理解了弦外之音
3. 每检测到一处，`subtext_instances` 计数 +1
4. 健康标准：有对话冲突的章节建议 ≥ 1 个潜台词实例

### 第六步: 对话推进检查

**`dialogue_advances_plot`**: 对话整体是否推进了以下至少一项：
- 剧情进展（新信息、决策、行动触发）
- 关系变化（信任/怀疑/亲近/疏远）
- 角色塑造（展示性格、暴露弱点、暗示秘密）

全章对话如果只是"闲聊"且不推进任何一项 → 标记为 `DIALOGUE_FLAT`

### 第七步: 生成报告

```markdown
# 对话质量检查报告

## 覆盖范围
第 {N} 章

## 对话统计
- 对话占比: {ratio}（{健康/偏低/偏高}）
- 对话轮数: {count}
- 说话角色数: {count}

## 辨识度检查
- 可辨识声音: {count} 个
- 无法区分角色对: {count} 对
  - {角色A} vs {角色B}：句式长度和语气相似度过高

## 意图层次
- 意图类型: {列表}
- 信息倾倒对话: {count} 处
  - 第{X}段: {角色}用对话解释{设定内容}，建议改为行动展示

## 独白控制
- 最长单人独白: {chars} 字（{角色}）
- 超限独白: {count} 处

## 对话推进性
- 推进剧情: {是/否}
- 推进关系: {是/否}
- 推进角色塑造: {是/否}

## 修复建议
- [辨识度] {角色A}与{角色B}对话风格过于相似，建议...
- [信息倾倒] 第{X}段改为带意图的对抗式对白
- [独白过长] {角色}的{X}字独白建议用动作/他人反应打断

## 综合评分
**对话质量**: {分数}/100
```

{
  "agent": "dialogue-checker",
  "chapter": 45,
  "overall_score": 82,
  "pass": true,
  "issues": [
    {
      "id": "DLG_001",
      "type": "DIALOGUE_FLAT",
      "severity": "medium",
      "location": "第12段",
      "description": "连续3轮对话仅传递背景信息，无角色意图冲突",
      "suggestion": "改为带意图的对抗式对白，加入试探/回避等层次",
      "can_override": false
    }
  ],
  "metrics": {
    "dialogue_ratio": 0.35,
    "info_dump_lines": 1,
    "subtext_instances": 3,
    "distinguishable_voices": 4,
    "indistinguishable_pairs": 0,
    "intent_types": ["试探", "施压", "回避"],
    "longest_monologue_chars": 120,
    "dialogue_advances_plot": true
  },
  "summary": "对话辨识度良好，意图层次丰富，无信息倾倒问题。"
}
```

**评分规则**: 使用统一扣分制公式（详见 `checker-output-schema.md` "统一评分公式"）：
- `overall_score = max(0, 100 - sum(deductions))`（critical=25, high=15, medium=8, low=3）
- `pass = overall_score >= 75`

> `issues[].type` 必须使用 `checker-output-schema.md` 定义的 13 个标准枚举值（本 checker 主要使用 `DIALOGUE_FLAT`、`DIALOGUE_INFODUMP`、`DIALOGUE_MONOLOGUE`、`PADDING`）。
```

## 禁止事项

❌ 通过全员标准书面语的对话且不预警
❌ 忽略超过 200 字的单人独白
❌ 接受连续 3+ 轮纯信息传递对话
❌ 对话辨识度检查只看内容不做遮名测试
❌ 忽略对话不推进剧情/关系/角色的问题

## 成功标准

- 可辨识声音数 ≥ 章内说话角色数的 80%
- 无法区分角色对数 = 0
- 信息倾倒对话 ≤ 1 处
- 最长独白 ≤ 200 字
- 对话推进剧情/关系/角色至少一项
- 意图类型 ≥ 2 种
- 对话占比在 0.25–0.55 健康范围内
