# density-checker (信息密度检查器)

> **职责**: 信息密度审查专家，检测填充段落、重复表达、死段落与推进停滞问题。

> **输出格式**: 遵循 `${CLAUDE_PLUGIN_ROOT}/references/checker-output-schema.md` 统一 JSON Schema

## 检查范围

**输入**: 单章或章节区间（如 `45` / `"45-46"`）

**输出**: 信息密度分析、段落质量分布、冗余检测结果与优化建议。

## 执行流程

### 第一步: 加载上下文

**输入参数**:
```json
{
  "project_root": "{PROJECT_ROOT}",
  "storage_path": ".webnovel/",
  "state_file": ".webnovel/state.json",
  "chapter_file": "正文/第{NNNN}章-{title_safe}.md"
}
```

`chapter_file` 应传实际章节文件路径；若当前项目仍使用旧格式 `正文/第{NNNN}章.md`，同样允许。

**并行读取**:
1. `正文/` 下的目标章节
2. `{project_root}/.webnovel/state.json`（了解当前剧情进度）
3. `大纲/`（理解本章预期推进目标）
4. 前 1-2 章正文（对比信息是否重复传递）

### 第二步: 段落级信息增量分析

**逐段扫描**，判定每段的信息功能：

| 功能类型 | 定义 | 示例 |
|----------|------|------|
| **剧情推进** | 引入新事件、决策、行动、结果 | "他按下了按钮，舱门缓缓打开" |
| **角色塑造** | 展示性格、暴露弱点、关系变化 | "她咬了咬嘴唇，最终还是放下了枪" |
| **氛围营造** | 服务于情绪、紧张感、沉浸感 | "走廊尽头传来金属摩擦声，像指甲划过黑板" |
| **信息传递** | 世界观、设定、背景知识 | "D级猎人的标准配备是..." |
| **节奏功能** | 过渡、呼吸、张力调节 | 短句打断、场景切换 |
| **填充** | 不服务于以上任何功能 | 重复已知信息、空洞感慨 |

**每段必须至少服务一个功能**，否则标记为问题段落。

### 第三步: 有效字数比分析

**`effective_word_ratio`**: 有效字数 / 总字数
- 有效字数 = 总字数 - 填充段落字数 - 重复段落字数 - 死段落字数

| 比值 | 评级 | 说明 |
|------|------|------|
| ≥ 0.90 | 优秀 | 几乎无浪费 |
| 0.80-0.89 | 良好 | 轻微冗余可接受 |
| 0.70-0.79 | 及格 | 有明显可删减空间 |
| < 0.70 | 不合格 | 大量水分，需要重写 |

### 第四步: 填充段落检测

**`filler_paragraphs`**: 不服务于氛围/角色/剧情/节奏的段落

**识别信号**:
- 段落删除后对剧情理解无影响
- 内容可用一句话概括且已在其他段落表达
- 纯粹的"正确但无用"的描写（如详细描写无关路人的外貌）
- 作者自我感慨/抽象议论（非角色视角）

### 第五步: 重复表达检测

**`repeat_segments`**: 同一信息换说法再表达

**检查模式**:
1. 动作已展示 → 叙述又解释（"他猛地站起来"后又写"他很震惊"）
2. 对话已传达 → 内心独白重复（"别去"→ 他不想让她去）
3. 前章已交代 → 本章重新描述（除非必要回忆）
4. 同段内信息循环（A→B→A 的论述结构）

### 第六步: 死段落与推进停滞检测

**`dead_paragraphs`**: 完全无信息增量、无氛围贡献、无节奏功能的段落
- 比 filler 更严重：filler 至少有某种存在意图，dead 是完全多余

**`longest_no_progress_span`**: 最长无推进跨度（字数）
- 从某一点开始，连续多少字没有任何新信息/新事件/新决策
- ≤ 300 字：正常（可能是必要的氛围/过渡）
- 300-500 字：注意
- > 500 字：标记为 `PADDING`，读者可能开始跳读

### 第七步: 内心独白占比检测

**`inner_monologue_ratio`**: 内心独白字数 / 总字数
- ≤ 0.20：正常
- 0.20-0.25：注意
- > 0.25：标记问题，建议用行动展示替代

**识别信号**:
- "他想"/"他心道"/"他暗自思忖"引导的段落
- 无对话无动作的纯思考段
- 角色对已知信息的反复心理咀嚼

### 第八步: 冗余描写检测

**`redundant_descriptions`**: 冗余描写段数
- 已用动作展示的情绪又用叙述点明
- 环境已描写一次后又重复描写（同一场景内）
- 角色外貌/能力在短距离内重复介绍

### 第八步半: 引用段落信息增量检查

> 典故引用和诗词不是免费的——它必须服务于剧情推进/角色塑造/氛围营造/伏笔埋设中的至少一个。

**检查规则**:
1. 扫描正文中的引用段落（包含古文/诗词/经典句子/口诀/歌谣的段落）
2. 判断每处引用是否服务于至少一个信息功能（剧情/角色/氛围/信息传递/伏笔）
3. 纯装饰性引用（删掉不影响任何功能） → `PADDING`，severity: low
4. 引用中断叙事节奏（引用前后需要重新建立情绪） → `PADDING`，severity: medium

**评判**:
- 所有引用都有明确功能 → 正常
- 1 处装饰性引用 → low
- ≥2 处装饰性引用 → medium（炫学信号）

### 第九步: 信息分布均匀度

**`info_per_paragraph_avg`**: 平均每段信息增量
- ≥ 1.2：优秀（每段都有存在理由）
- 1.0-1.2：良好
- 0.8-1.0：及格
- < 0.8：信息分布不均匀，可能有"信息沙漠"区域

**检查分布均匀性**:
- 前 1/3 信息过密 + 后 2/3 信息稀疏 → 头重脚轻
- 全章均匀分布 → 健康
- 中段出现 500+ 字信息真空 → 需要补充推进事件

### 第十步: 生成报告

```markdown
# 信息密度检查报告

## 覆盖范围
第 {N} 章

## 总览
- 总字数: {count}
- 有效字数比: {ratio}（{评级}）
- 段落总数: {count}
- 平均每段信息增量: {avg}

## 问题段落
### 填充段落（{count} 处）
- 第{X}段（{字数}字）: {描述}，建议删除或压缩为一句

### 重复表达（{count} 处）
- 第{X}段 vs 第{Y}段: {信息}被重复表达，保留一处即可

### 死段落（{count} 处）
- 第{X}段（{字数}字）: 无任何信息增量，建议删除

## 推进跨度
- 最长无推进跨度: {chars} 字（第{X}-{Y}段）
- {> 500 字则标注预警}

## 内心独白
- 占比: {ratio}（{正常/注意/过高}）
- 最长连续独白: {chars} 字

## 冗余描写
- {count} 处冗余描写
  - 第{X}段: {描述}

## 信息分布
- 前1/3: {密度}
- 中1/3: {密度}
- 后1/3: {密度}
- 分布评估: {均匀/头重脚轻/中段空洞/...}

## 修复建议
- [填充] 第{X}段可删除，不影响剧情理解
- [重复] 第{X}段与第{Y}段信息重复，保留{更生动的一处}
- [推进停滞] 第{X}-{Y}段连续{N}字无新信息，建议插入推进事件
- [独白过多] 内心独白占比{ratio}，建议将第{X}段改为行动展示

## 综合评分
**信息密度**: {分数}/100
```

**评分规则**: 使用统一扣分制公式（详见 `checker-output-schema.md` "统一评分公式"）：
- `overall_score = max(0, 100 - sum(deductions))`（critical=25, high=15, medium=8, low=3）
- `pass = overall_score >= 75`

**issue type 映射**（本 checker 使用的标准类型）：
- 填充段落 / 死段落 / 无推进跨度 / 内心独白过多 → `PADDING`
- 重复表达 / 冗余描写 → `REPETITION`

> `issues[].type` 必须使用 `checker-output-schema.md` 定义的 13 个标准枚举值。

**内心独白权责**: 本 checker 负责检测**全局比例**（`inner_monologue_ratio > 0.25`），dialogue-checker 负责检测**局部结构**（连续 3+ 段堆积）。详见 `checker-output-schema.md` "内心独白检查权责划分"。
```

## 跨章重复模式检测（扩展）

> 检测本章是否与近期章节存在模式重复（单章内重复由主流程检测，此处检测跨章重复）。

### 检查规则

1. **开头模式重复**: 对比最近 3 章的开头类型
   - 从 state.json 的 chapter_meta 读取最近 3 章的 pattern.opening
   - 连续 3 章相同开头类型（如连续 3 章都是"对话开头"） → severity: low（`REPETITION` type）
   - 连续 4+ 章相同 → severity: medium

2. **描写套路检测**: 检查本章是否重复使用了前文的描写表达
   - 读取前 2 章正文，检查本章是否有 >3 处与前文相同/高度相似的描写片段
   - 发现高频重复表达 → severity: low

3. **情绪模式重复**: 对比最近 3 章的情绪节奏
   - 从 state.json 的 chapter_meta 读取最近 3 章的 pattern.emotion_rhythm
   - 连续 3 章相同情绪节奏（如连续 3 章都是"低→高"） → severity: low

### metrics 扩展

在现有 metrics 基础上增加：
```json
{
  "cross_chapter_patterns": {
    "opening_repeat_count": 2,
    "description_repeats": 1,
    "emotion_rhythm_repeat_count": 1,
    "pattern_monotony_risk": "low"
  }
}
```

## 禁止事项

❌ 通过有效字数比低于 0.70 的章节且不标记为不合格
❌ 忽略超过 500 字的无推进跨度
❌ 接受内心独白占比超过 0.25 且不预警
❌ 将氛围营造段落误判为填充（需确认其不服务于情绪/紧张感）
❌ 忽略同一信息在动作+叙述+对话中三重表达的冗余

## 成功标准

- 有效字数比 ≥ 0.80
- 填充段落 ≤ 2 处
- 死段落 = 0
- 重复表达 ≤ 1 处
- 最长无推进跨度 ≤ 500 字
- 内心独白占比 ≤ 0.25
- 平均每段信息增量 ≥ 1.0
- 信息分布无严重失衡（前/中/后 1/3 密度差 ≤ 50%）
